# Development Phases Plan
## Phase 0 — Foundation (1-2 weeks)
- Monorepo/project scaffolding
- Core architecture skeleton
- Provider adapter interfaces
- Baseline desktop shell (Qt6)
- Logging + config framework

## Phase 1 — Discovery and Spec Engine (2-3 weeks)
- Prompt intake UI
- Clarification wizard with checkbox/panel UX
- Spec generator agent (PRD/TRD/DEV_PHASES/DESIGN)
- Review agent pass for realism and completeness
- User approval flow (notify-only vs auto-continue)

## Phase 2 — Orchestrated Build Loop (3-5 weeks)
- Coding agent phase execution engine
- Task decomposition and phase planner
- Code generation + commit checkpoints
- Test/lint integration

## Phase 3 — Deep Review and Security (2-4 weeks)
- Code reviewer agent with policy checks
- Vulnerability tester (deps + static analysis)
- Fix suggestion/auto-fix loop
- Regression prevention checks

## Phase 4 — Production Readiness (2-3 weeks)
- Launch checklist automation:
  - auth correctness
  - legal pages/privacy/terms
  - cookie consent
  - monitoring/error tracking
- Release readiness scorecard

## Phase 5 — SEO and Analytics (1-2 weeks)
- SEO agent keyword workflow
- Metadata/schema suggestions
- GA4/Search Console integration tasks
- Guided manual setup fallback docs

## Phase 6 — Handover and DX (1-2 weeks)
- Auto README and architecture docs
- Deployment runbook
- Maintenance checklist
- Project handover bundle export

## Phase 7 — Reliability, Learning, and Scale (ongoing)
- Run-memory and error pattern learning (policy-safe)
- Better retry/recovery heuristics
- Cross-project best-practice extraction
- Extended MCP/skill ecosystem support

## Cross-Cutting Workstreams
- Performance optimization
- Security hardening
- UX polish (modern minimal design language)
- Provider regression testing (NVIDIA/OpenRouter required)
- Documentation and examples

## Definition of Done (Global)
- All phase gates passed
- Critical/high vulnerabilities resolved
- Production checklist complete
- User-approved handover package delivered
